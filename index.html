모든 메뉴를 함께 동기화하는 로그인

로그인하면 일기, 메모, 가계부, 기록, 프로필 전체가 같은 계정의 Firestore 문서 하나에 저장됩니다.

동기화 상태

로컬 저장 모드
사진은 데이터에 함께 저장되므로 큰 사진을 많이 첨부하면 브라우저 저장공간이나 클라우드 저장 제한에 걸릴 수 있습니다. 작은 용량의 사진을 권장합니다.

Firebase 설정값이 비어 있으면 현재 브라우저 localStorage에 저장됩니다.

계정으로 로그인 중입니다.

Firebase 설정 위치

아래 코드의 firebaseConfig 부분에 Firebase 웹 앱 설정값을 넣으면 로그인/동기화가 작동합니다.

const firebaseConfig = { apiKey: "여기에 입력", authDomain: "여기에 입력", projectId: "여기에 입력", storageBucket: "여기에 입력", messagingSenderId: "여기에 입력", appId: "여기에 입력" };
HTML에 직접 비밀번호를 넣지 마세요. 이메일/비밀번호 관리는 Firebase Authentication이 담당해야 합니다.

Firestore 보안 규칙 예시

로그인한 사용자가 자기 데이터만 읽고 쓰도록 제한하는 예시입니다.

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{userId}/homepage/{docId} { allow read, write: if request.auth != null && request.auth.uid == userId; } } }
